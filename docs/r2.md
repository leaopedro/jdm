# Cloudflare R2 — bucket CORS runbook

The mobile and admin clients PUT directly to R2 using a server-issued
presigned URL. R2 enforces CORS on the bucket; the API's
`CORS_ORIGINS` setting only controls the API's own CORS, **not** R2.

If browser uploads fail with:

```
Access to fetch at 'https://<account>.r2.cloudflarestorage.com/...' from origin
'https://<app>' has been blocked by CORS policy: Response to preflight request
doesn't pass access control check: No 'Access-Control-Allow-Origin' header is
present on the requested resource.
```

…the bucket CORS rules are missing or do not include the requesting origin.

## Canonical CORS spec

The canonical, repo-tracked CORS spec lives in `infra/r2-cors.json`. Apply it
to **every** bucket that receives presigned PUTs (prod + preview).

## Apply via Cloudflare dashboard

1. Cloudflare → R2 → select the bucket.
2. **Settings** → **CORS Policy** → **Add CORS policy** (or Edit).
3. Paste the contents of `infra/r2-cors.json`.
4. Save. Changes propagate within seconds.
5. Hard-reload the affected client and retry the upload.

## Apply via wrangler

```bash
# auth (one-time on the operator machine)
wrangler login

# apply
wrangler r2 bucket cors put <BUCKET_NAME> --rules infra/r2-cors.json

# verify
wrangler r2 bucket cors list <BUCKET_NAME>
```

Run once for the prod bucket and once for the preview bucket. Bucket names are
in `R2_BUCKET` (Railway service variables).

## Apply via S3 API (fallback)

Uses the `s3api put-bucket-cors` shape with R2's S3-compatible endpoint.

```bash
aws s3api put-bucket-cors \
  --endpoint-url "https://${R2_ACCOUNT_ID}.r2.cloudflarestorage.com" \
  --bucket "${R2_BUCKET}" \
  --cors-configuration "$(jq '{CORSRules: .}' infra/r2-cors.json)"
```

## Adding a new origin

Edit `infra/r2-cors.json`, open a PR, then re-apply with one of the methods
above. Do not edit dashboard rules in place without updating the file —
config drift will recur on the next bucket recreation.

## Verifying

After applying, run from the affected origin:

```bash
curl -i -X OPTIONS \
  -H 'Origin: https://<your-origin>' \
  -H 'Access-Control-Request-Method: PUT' \
  -H 'Access-Control-Request-Headers: content-type' \
  "https://<account>.r2.cloudflarestorage.com/<bucket>/probe"
```

Expect `HTTP/1.1 200` (or `204`) with `Access-Control-Allow-Origin` echoing
your origin and `Access-Control-Allow-Methods` containing `PUT`.
